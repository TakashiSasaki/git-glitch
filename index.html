Note

In the Resource Description Framework (RDF), a triple is a basic data structure composed of three parts: Subject, Predicate, and Object. It represents relationships between resources.

On the other hand, a quad is a data structure composed of four parts, adding context or graph name to a triple. The components of a quad are as follows:

  1. Subject: The resource that is the subject of the relationship.
  2. Predicate: The resource representing the relationship between the subject and object.
  3. Object: The resource that is the object of the relationship.
  4. Graph Name or Context: The identifier of the RDF graph to which this triple belongs. This part allows for the reuse of the same triple in different contexts or graphs.

The introduction of quads makes the modeling of RDF data more flexible. By explicitly specifying to which graph a particular triple belongs, it becomes possible to effectively manage different datasets or views, security contexts, and more.
